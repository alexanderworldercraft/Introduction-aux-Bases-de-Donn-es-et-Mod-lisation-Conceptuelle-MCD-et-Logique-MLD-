Introduction aux Bases de Données et Modélisation Conceptuelle (MCD) et Logique (MLD)

Qu'est-ce qu'une base de données ?

Une base de données (BDD) est un ensemble organisé de données structurées, généralement stockées et accessibles électroniquement depuis un système informatique. Les bases de données sont essentielles pour stocker, gérer et récupérer efficacement des données.

Les modèles conceptuels et logiques

  • Modèle Conceptuel des Données (MCD) :
    • Représente les concepts et les relations entre eux dans le système sans se préoccuper des aspects techniques.
    • Utilise des entités, des associations et des attributs pour décrire les données.
  • Modèle Logique des Données (MLD) :
    • Traduit le MCD en un schéma plus technique qui peut être implémenté dans un SGBD (Système de Gestion de Base de Données).
    • Utilise des tables, des colonnes et des clés pour structurer les données.

Exemple Concret : Application Web d'Archives de Documents

Nous allons créer une base de données pour une application web d'archives de documents avec les tables et colonnes suivantes :

Structure de la Base de Données : Archive

Tables :

  • Utilisateur
    • UtilisateurID (clé primaire)
    • EtatID (clé étrangère)
    • Nom (varchar(50))
    • MotDePasse (varchar(255))
    • Telephone (varchar(20))
  • Document
    • DocumentID (clé primaire)
    • EtatID (clé étrangère)
    • Titre (varchar(50))
    • Description (text)
  • Tag
    • TagID (clé primaire)
    • EtatID (clé étrangère)
    • Nom (varchar(50))
  • DocumentTag
    • DocumentID (clé étrangère)
    • TagID (clé étrangère)
  • Fichier
    • FichierID (clé primaire)
    • DocumentID (clé étrangère)
    • EtatID (clé étrangère)
    • Chemin (text)
  • Etat
    • EtatID (clé primaire)
    • Nom (varchar(50))

Modèle Conceptuel des Données (MCD)

Le MCD représente les entités et leurs relations :

  • Entités : Utilisateur, Document, Tag, DocumentTag, Fichier, Etat
  • Relations :
    • Un Utilisateur a un Etat.
    • Un Document a un Etat.
    • Un Tag a un Etat.
    • Un Document peut avoir plusieurs Tags (relation DocumentTag).
    • Un Document peut avoir plusieurs Fichiers.
    • Un Fichier a un Etat.

Modèle Logique des Données (MLD)

Le MLD traduit le MCD en schéma relationnel :

copy code 
CREATE DATABASE Archive;

USE Archive;
                    
CREATE TABLE Etat (
    EtatID INT AUTO_INCREMENT PRIMARY KEY,
    Nom VARCHAR(50) NOT NULL
);
                    
CREATE TABLE Utilisateur (
    UtilisateurID INT AUTO_INCREMENT PRIMARY KEY,
    EtatID INT,
    Nom VARCHAR(50) NOT NULL,
    MotDePasse VARCHAR(255) NOT NULL,
    Telephone VARCHAR(20),
    FOREIGN KEY (EtatID) REFERENCES Etat(EtatID)
);
                    
CREATE TABLE Document (
    DocumentID INT AUTO_INCREMENT PRIMARY KEY,
    EtatID INT,
    Titre VARCHAR(50) NOT NULL,
    Description TEXT,
    FOREIGN KEY (EtatID) REFERENCES Etat(EtatID)
);
                    
CREATE TABLE Tag (
    TagID INT AUTO_INCREMENT PRIMARY KEY,
    EtatID INT,
    Nom VARCHAR(50) NOT NULL,
    FOREIGN KEY (EtatID) REFERENCES Etat(EtatID)
);
                    
CREATE TABLE DocumentTag (
    DocumentID INT,
    TagID INT,
    PRIMARY KEY (DocumentID, TagID),
    FOREIGN KEY (DocumentID) REFERENCES Document(DocumentID),
    FOREIGN KEY (TagID) REFERENCES Tag(TagID)
);
                    
CREATE TABLE Fichier (
    FichierID INT AUTO_INCREMENT PRIMARY KEY,
    DocumentID INT,
    EtatID INT,
    Chemin TEXT NOT NULL,
    FOREIGN KEY (DocumentID) REFERENCES Document(DocumentID),
    FOREIGN KEY (EtatID) REFERENCES Etat(EtatID)
);

Explications détaillées

  • Etat :
    • Gère les différents états possibles pour les autres entités (corbeille, supprimé, etc.).
  • Utilisateur :
    • Chaque utilisateur est associé à un état.
    • Contient des informations basiques comme le nom, mot de passe et téléphone.
  • Document :
    • Stocke les informations principales des documents.
    • Associé à un état pour la gestion des statuts.
  • Tag :
    • Permet de classer les documents en utilisant des mots-clés.
    • Chaque tag est aussi associé à un état.
  • DocumentTag :
    • Table de liaison pour les relations plusieurs-à-plusieurs entre les documents et les tags.
  • Fichier :
    • Stocke les chemins des fichiers associés aux documents.
    • Chaque fichier a un état et est lié à un document.

Conclusion

Ce schéma fournit une base solide pour comprendre la modélisation des bases de données en utilisant MCD et MLD. En suivant cet exemple, un étudiant peut comprendre comment concevoir une base de données relationnelle et la mettre en œuvre efficacement.
